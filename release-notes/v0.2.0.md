# v0.2.0

**Release Date:** 2026-02-08

Multi-scene workflows, scene transitions, and comprehensive documentation. This release adds the ability to organize animations into multiple scenes with configurable transitions, a multi-scene sequencer for playback, and full documentation accessible directly from the editor.

## Highlights

- Multiple scenes per project with independent elements and timelines
- Scene transitions (fade, slide) with visual configuration dialog
- TinyflySequencer for multi-scene playback with transitions
- Comprehensive documentation with in-app viewer at `/docs`

---

## Added

### Multiple Scenes

- **Scene management** - Add, rename, duplicate, reorder, and delete scenes via the scene bar
- **Independent elements & timelines** - Each scene has its own set of elements and animation timeline
- **Scene persistence** - Scenes are saved and restored with project auto-save

### Scene Transitions

- **Transition types** - `none`, `fade`, `slide-left`, `slide-right`, `slide-up`, `slide-down`
- **Transition dialog** - Visual configuration with type dropdown, duration input, and live preview
- **Scene bar indicators** - Clickable transition icons between scene tabs
- **Context menu** - Right-click a scene tab to set transitions
- **Editor preview** - CSS transition animation plays when switching scenes in the editor

### Multi-Scene Sequencer

- **TinyflySequencer class** - Plays scenes in order with transitions between them
- **Dual-container architecture** - Both outgoing and incoming scenes render simultaneously during transitions
- **Playback controls** - `play()`, `pause()`, `stop()`, `goToScene(index)`, `destroy()`
- **State tracking** - `currentSceneIndex`, `sceneCount`, `isPlaying`, `state` properties
- **Callbacks** - `onComplete` and `onSceneChange` event handlers
- **playSequence() helper** - Quick function to create and play a sequence
- **30 new tests** for the sequencer

### Multi-Scene Embed

- **Scope toggle** - "Single Scene" vs "All Scenes (Sequence)" in the embed dialog
- **Sequence code generation** - Generates embed code using `tinyfly.playSequence()`
- **Inline and external file** options for sequence JSON

### Documentation

- **Getting Started guide** - Installation, key concepts, first animation walkthrough
- **Editor Guide** - Complete visual editor reference covering all features, 20 presets, 42 samples, keyboard shortcuts
- **API Reference** - Full documentation for Timeline, Track, Clock, Easing, Interpolators, Player, Sequencer, Adapters, and Export formats
- **Code Examples** - 12 practical examples (fade, multi-property, looping, color, canvas, SVG, sequences, motion path, embedding)

### In-App Documentation Viewer

- **`/docs` route** - Dedicated page with sidebar navigation and rendered markdown content
- **Lightweight markdown renderer** - Custom built, zero dependencies
- **Bundled at build time** - Docs imported via Vite `?raw`, works offline
- **Dark theme** - Matches the editor UI
- **"Docs" toolbar button** - Quick access from the editor

### Engine & Types

- **SequenceDefinition type** - JSON-serializable format for multi-scene animations
- **SceneTransition type** - Transition configuration (`type` + `duration`)
- **TransitionType** - Union type for all transition effects
- **SerializedElement** - Minimal element representation for the sequencer
- **DEFAULT_TRANSITION** - Default no-transition constant
- **exportSequence()** - ProjectStore method to build a SequenceDefinition from the current project

## Changed

- **Embed dialog** - Redesigned with scope selector for single scene vs all scenes
- **Scene bar** - Enhanced with transition indicators between tabs and context menu additions
- **Project store** - Extended with `setSceneTransition()`, `getSceneTransition()`, and `exportSequence()` methods
- **Scene types** - Added optional `transition` field to `SceneDefinition`
- **Player exports** - Added sequencer, sequence types, and transition exports
- **Editor preview** - Added `overflow: hidden` and transition animation CSS
